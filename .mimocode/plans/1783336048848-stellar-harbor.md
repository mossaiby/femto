# Plan: Real Code Generation in Main Driver

## Current State

The main driver (`src/main.cpp`) only lexes and dumps tokens. The full pipeline exists in pieces:
- Lexer, Parser, Sema, AST->HIR, HIR Optimizer, LIR Optimizer, x86-64 Codegen
- **Missing**: HIR-to-LIR lowering (no `src/lir/hir_to_lir.*` files)
- **Missing**: NASM/assembler/linker invocation
- `std::io::print` and `std::string8::from_int` are extern declarations — need runtime stubs

## Design: Runtime Strategy

The user wants most stdlib written in femto itself. For OS calls, embed minimum syscall stubs as `__builtin_XXX()` in codegen, callable from femto std.

For this initial implementation:
- The codegen recognizes `__builtin_write(fd, buf, len)` and `__builtin_exit(code)` and emits raw Linux syscalls
- The femto `std::io` functions (`print`, `println`) are declared as extern but implemented by the codegen emitting NASM stubs that call syscalls directly
- `std::string8::from_int` gets a simple stub too (convert int to decimal string)
- This lets us run hello_world and factorial end-to-end without requiring the full stdlib to compile first

## Changes

### 1. Create HIR-to-LIR Lowering (`src/lir/hir_to_lir.h` + `src/lir/hir_to_lir.cpp`)

New class `HIRToLIR` lowering `HIRModule` → `LIRModule`:
- Walk HIR functions → LIR functions with basic blocks
- Lower HIR statements: `AssignStmt`→`Store`, `ReturnStmt`→`Ret`, `IfStmt`→`Branch`+`Jump`, `WhileStmt`→loop labels
- Lower HIR expressions: `IntLit`/`VarRef`/`BinaryOp`/`UnaryOp`/`CallOp`/`StringLit`/`BoolLit`
- Binary arithmetic ops → LIR `Add/Sub/Mul/Div/Mod`
- Comparisons → `CmpEq`..`CmpGe` + `setXX`
- Logical ops → `LogicAnd`/`LogicOr`
- String literals → tracked for data section emission
- Virtual register allocation via `LIRFunction::new_reg()`

### 2. Rewrite `src/main.cpp` — Full Pipeline + CLI Flags

New CLI:
```
femto [options] <source.femto>

Options:
  -o <file>           Output executable path (default: <stem>)
  --emit-asm <file>   Write NASM assembly to file
  --emit-obj <file>   Write object file to file
  --keep-asm          Keep intermediate .asm (default: deleted)
  --keep-obj          Keep intermediate .o (default: deleted)
  --emit-hir          Dump HIR to stderr
  --emit-lir          Dump LIR to stderr
  --tokens            Dump token stream to stderr
  --ast               Dump AST to stderr
  --no-codegen        Stop after type checking
  -v, --verbose       Print pipeline progress
  -h, --help          Show usage
```

Pipeline:
```
Read source → Lex → Parse → TypeCheck → AST→HIR → HIR Opt → HIR→LIR → LIR Opt → Codegen (NASM) → nasm -f elf64 → ld → executable
```

NASM/LD invocation via `std::system()`:
- `nasm -f elf64 <file>.asm -o <file>.o`
- `ld -o <output> <file>.o` (bare-metal style with syscall stubs, no libc)

### 3. Enhance x86-64 Codegen (`src/codegen/x86_64/x86_64.cpp`)

- **String data section**: Emit string literals in `.data` section with `db` directives
- **Syscall stubs**: For known extern functions (`std.io.print`, `std.io.println`, `std.string8.from_int`), emit NASM functions using Linux syscalls:
  - `std.io.print`: `write(1, buf, len)` via `syscall` with `rax=1`
  - `std.io.println`: same + append `\n`
  - `std.string8.from_int`: small int-to-string converter using repeated division
  - `__builtin_exit`: `exit(code)` via `syscall` with `rax=60`
- **Entry point**: `_start` calls `main`, passes argc/argv via stack, exits with return value
- **Function calls**: System V AMD64 ABI (rdi, rsi, rdx, rcx, r8, r9 for first 6 params)

### 4. Update `src/CMakeLists.txt`

Add `lir/hir_to_lir.cpp` to both `femto` and `femto_tests` targets.
